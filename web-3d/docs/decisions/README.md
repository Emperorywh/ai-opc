# 架构决策记录（ADR）索引

## 用途

本项目的重要决策目前分散在两份**已定稿**文档中。本索引统一登记其位置，避免散落难寻。**不重复**决策正文，只指向出处。

## 未来新增 ADR 规则

实施过程中若出现**新的、需要长期留痕的架构 / 技术决策**，用独立 ADR 文件记录：

- **命名**：`0001-<kebab-slug>.md`，序号从 `0001` 递增（与 D/R 系列独立计数）。
- **时机**：高风险变更（见 `CLAUDE.md §3`）、或推翻 D/R 系列既有决策时。
- **不记**：具体 Task 的实现细节（那是 `docs/PROGRESS.md` 的事）。

> 现有 D/R 决策若被未来 ADR 推翻，新 ADR 在「状态」里标注「取代 D? / R?」，并在上方对应表格行补注。

### ADR 模板骨架

````md
# 0001 · <决策标题>

| 项 | 值 |
|---|---|
| 状态 | 提议 / 已接受 / 已废弃 / 已被 00XX 取代 |
| 日期 | YYYY-MM-DD |
| 影响 | <受影响的子系统 / Milestone> |

## 背景
（为什么需要这个决策？遇到了什么问题？）

## 决策
（决定了什么？）

## 理由
（为什么这么选？权衡了哪些备选方案？引用相关的 D/R 决策。）

## 后果
（正面 / 负面影响，需配套的改动或迁移。）
````
