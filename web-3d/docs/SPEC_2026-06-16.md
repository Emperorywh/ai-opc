## 1. 项目概述与目标

构建一个运行于 Web 浏览器的**动漫风格、科技感、平铺式**交互 3D 世界地图。区别于传统 GIS / 卫星地图：

- **固定倾斜视角（Tilt View）**，非可自由旋转的地球，呈现"沙盘"观感。
- 经过艺术化处理的平面世界地图，但具备**真实地理数据与三维地形起伏**。
- 面向**教育展示与可视化**场景。

### 视觉风格配比（硬约束）

```
50% 动漫风格  +  30% 手绘地理图鉴  +  20% 科技感可视化
```

**避免**：Google Earth 写实风、ArcGIS 风、高德/百度地图风、写实 PBR、强烈镜面反射、锯齿感、写实岩石贴图、生硬 GIS 边界。

**最终关键词**：Stylized World Map · Anime Inspired · Hand Painted Terrain · Educational Sandbox · Civilization VI Inspired · Low Saturation · Soft Lighting · Tilted Camera · Fantasy Atlas · Interactive Geography · Museum Quality Visualization · High-end WebGL Experience。

## 2. 视觉风格规范

### 2.1 色彩规范（低饱和、水彩感）

| 地物 | 颜色方向 | 参考 |
|---|---|---|
| 草地/平原 | 鼠尾草绿 | `#8FA98A`～`#A9C0A0` |
| 山脉 | 暖灰绿 | `#7E8B76`～`#9AA892` |
| 高原 | 沙橄榄 | `#A39A78` |
| 沙漠 | 浅赭石 | `#D9C39B`～`#C9B083` |
| 海洋（浅） | 青绿 | `#7FC4C0` |
| 海洋（深） | 深青绿渐变 | `#2E6E73` |
| 河流 | 青蓝发光 | `#6FD0E8` |
| 边界描边 | 柔光暖白 | `#F3E9D2`（半透明） |
| 争议边界 | 虚线暖灰 | `#C9BFA8` |

> 所有色值需经"低饱和化"统一处理（S 降 15–25%），整体保持水彩插画一致性。

### 2.2 着色策略（D6：程序化水彩 + 软描边）

自定义 ShaderMaterial，输入：**归一化高度** `h`、**坡度** `slope`（由法线 y 分量）、**纬度**、**噪声纹理**。

1. **高度分区分层上色**：用 `smoothstep` 在多个高度带间过渡（海岸→沙滩→平原→丘陵→山脉→雪线），每带取上表色，带间加 1–3 px 软过渡（非硬边）。
2. **水彩噪声叠加**：用低频 value/simplex 噪声调制颜色明度与微小花斑，形成"水彩颗粒"质感；同高度区域不显得平。
3. **坡度强调**：陡坡（山脉侧）偏暖灰绿，缓坡偏草绿；强化山体立体感但不写实。
4. **软描边（轮廓线）**：在法线与视线接近垂直处（`1 - dot(N,V)`）叠加轻微暖白描边，类似动画电影场景描边；强度可调。
5. **高度夸张 ×2.5**（见 §6.1）。

> 不使用 MeshStandardMaterial 的 PBR 流程；用自定义 Lambert-like 漫反射 + 环境光主导，避免镜面高光。

### 2.3 光照（温暖日照、环境光主导）

- **单一方向光**：暖白（`#FFF2D8`），固定俯角约 50°，主要用于坡向着色；**不启用实时 shadow map**（避免写实感与高开销），"柔和投影感"通过坡度/高度驱动的接触阴影 shader 近似，按质量档可关。
- **环境光**：半球光（天空 `#E8F0F2` / 地面 `#9C9078`），强度为主，整体偏亮、低对比。
- **无点光源/无聚光**，避免写实感。
- 固定光照（不实现昼夜变化；如未来需要，预留 `sunDirection` uniform）。

### 2.4 描边规则（重要元素）

- 国家边界、河流、海岸线：**轻微描边**（动画电影场景质感），半透明暖白，宽度随缩放微调，避免纯黑硬线。
- 海岸线由地形高度 ≈ 0 的等值线自动生成（shader 内基于 `h` 的 `fwidth` 抗锯齿等高线），非额外几何。

## 5. 坐标系与投影（D4 推荐）

### 5.1 工作坐标系

世界平面尺寸固定：**宽 2.0（X）× 纵 1.0（Z）**（Z 为地图平面的纵向/南北方向，非 3D 高度轴 Y），对应 equirectangular 的 2:1 经纬比。高度（Y）独立，按真实高度 × **2.5**（见 §6.1）。

```
x = lon / 180 * 1.0        // lon ∈ [-180,180] → x ∈ [-1,1]
z = -lat / 90 * 0.5         // lat ∈ [-90,90]  → z ∈ [-0.5,0.5]（向北 -z）
y = heightExaggerated
```

所有子系统（地形顶点、边界顶点、河流、标签锚点）必须经过**同一投影函数** `project(lon,lat) → (x,z)`，确保对齐。

### 5.2 投影分阶段策略（权衡说明）

> 你要求"美学优先，作者推荐"。但完全的美学投影（Robinson/Winkel-Tripel）会让 DEM 不能 1:1 采样、所有 GeoJSON 需重投影。鉴于零预算离线烘焙（D16）且要求快速 MVP（D15），采取**分阶段**策略，工程可落地且不牺牲长期美学：

| 阶段 | 投影 | 理由 |
|---|---|---|
| **MVP（Phase 1）** | **Equirectangular（等距柱状）** | DEM（Copernicus）1:1 直接作为 heightmap 纹理；边界/河流投影最简；GPU 位移即用；快速跑通沙盘观感与性能基线。**已知缺陷**：高纬横向拉伸，南极洲显示偏宽。 |
| **Phase 2（美学升级）** | **Robinson 或 Winkel-Tripel** | 离线**一次性**重投影 DEM 与所有矢量数据并重新烘焙（成本一次性、可接受），获得"国家地理图鉴"美感，同时解决极区拉伸。 |

**架构保证**：投影被封装在 `config/projection.ts` 的单一函数与 uniform，切换投影只需改一处 + 重跑 pipeline，渲染层零改动。MVP 即便用 equirectangular，也可通过**纬度纵向压缩掩码**缓解南极拉伸观感。

## 6. 核心子系统设计

### 6.1 地形系统（D5 推荐：GPU 顶点位移 + 静态密度掩码）

**真实高程 + 轻微夸张**：真实高度 × 2.5，层次更明显（需求要求 2~3 倍）。

**网格方案（推荐，非流式）**：

1. 基础网格：一张平面（PlaneGeometry），细分密度按性能预算（如 512×256 ~ 2048×1024，按质量档缩放）。
2. **顶点位移**：在顶点着色器中采样 `heightmap.png`（equirect，16-bit），按 §5.1 映射，输出 `y`。**省顶点数、性能好**。
3. **细节增强**：片元着色器采样 `normal.png`（烘焙法线贴图）补充亚像素细节，避免低频地形发软。
4. **静态密度掩码（可选增强）**：对山脉密集区（喜马拉雅/安第斯/落基/阿尔卑斯）在烘焙期提高局部网格细分（预生成非均匀网格），其余区域低密度。**仅静态一次**，不做运行时 LOD 流式。
5. **流式四叉树 LOD** 明确列为 Phase 3+ 候选，仅当缩放细节需求证明必要时再做（含 skirts 接缝缝合 / geomorphing）。

**关键地物覆盖**（烘焙进 heightmap，自然呈现）：喜马拉雅、青藏高原、安第斯、落基、阿尔卑斯、格陵兰、南极冰盖。七大洲/四大洋/主要岛屿/海岸线由 DEM 与海洋系统共同界定。

**陷阱**：位移用**自定义 uniform**（如 `uHeightScale`/`uHeightOffset`）统一控制（自定义 ShaderMaterial 无内置 `displacementScale`/`displacementBias`）；CPU 侧需一份**精简高度查询表**供河流采样与拾取高亮深度偏移用（避免 CPU/GPU 不一致）。

### 6.2 海洋系统（D8：Gerstner 波 + 菲涅尔 + 深度渐变）

自定义 ShaderMaterial，**非纯色背景**，具备体积感/波纹/柔和反射/深浅渐变：

1. **Gerstner 波**：叠加 3–5 个不同方向/频率的 Gerstner 波，顶点位移产生柔和浪涌；强度按质量档降级（低档减为正弦波）。
2. **菲涅尔柔和反射**：`fresnel = pow(1 - dot(N,V), 3)`，掠射角偏亮青绿，模拟柔和反射（非镜面）。
3. **深浅渐变**：用 heightmap 采样的**水深**（地形高度低于海平面越多越深）在浅青绿 → 深青绿间插值，近岸浅、远洋深，形成层次。
4. **流动**：时间 uniform 驱动波纹与一组细微法线纹理滚动，呈现流动感。
5. **体积感**：海平面略低于地形 0 高度，海岸线处 heightmap 自然过渡，配合轻微雾化增强纵深。

**性能**：透明物体**后绘**、关闭深度写入、按深度排序（Terrain 先绘写深度）；波数随 `AdaptiveQuality` 调整。

### 6.3 国家边界系统（D9 颜色拾取 + D10 争议虚线）

**风格**：发光描边、柔和轮廓线、非 GIS 硬边界。

**结构**：

- **CountryMeshes（填充面）**：每个国家一个三角化多边形（`earcut`），材质为半透明低饱和填充（默认几乎不可见）。**两层用途**：
  - **拾取层**：离屏 RT 用唯一颜色 ID 渲染（见下）。
  - **高亮层**：hover/selected 时填充色提亮 + 边缘发光。
- **BorderLines（描边）**：边界线段（LineSegments / Line2），暖白半透明，柔和；海岸线由地形 shader 等高线自动生成，不在此重复。
- **DisputedLines（争议虚线）**：用 `ne_10m_admin_0_boundary_lines_disputed_areas`，`LineDashedMaterial` 暖灰虚线渲染。

**GPU 颜色拾取（D9）实现**：

1. 维护 `countryId ↔ RGB` 映射（`id → (r,g,b)`，`id = r<<16|g<<8|b`）。
2. 离屏 RenderTarget（低分辨率即可，如 1:1 指针像素读取），用纯 ID 颜色 Material 渲染所有国家面。
3. `onPointerMove`（节流）→ 读取指针下 1×1 像素 → 反查 id → `store.setHovered(id)`。
4. 点击同理 → `store.setSelected(id)`。
5. 优化：仅在指针移动且命中测试变化时重渲拾取 RT；或每帧低分辨率重渲（成本低）。

> 替代方案（BVH / 2D 点在多边形内）已评估，颜色拾取对"数百国家、每个上千顶点"最稳，O(1)。

**争议边界合规（D10）**：默认 Natural Earth 主流立场 + 争议区虚线，教育中立；`config` 预留 `boundaryVariant` 接口，未来可切换数据源（中文版/国际版）。**克什米尔、克里米亚、西撒哈拉、台湾、巴勒斯坦、阿布哈兹等**均按 NE 默认，虚线表达争议。

### 6.4 河流系统（D12：DEM 采样带状几何 + 流动 shader）

覆盖：长江、黄河、亚马逊、尼罗河、密西西比、多瑙等主要河流。

**实现（贴地、不穿山、无 z-fighting）**：

1. **离线采样**：pipeline 中将简化后的河流折线**按 DEM 重采样高度**——每个折点查询烘焙 heightmap 得到地表 `y`，再加固定偏移 `+ε`（如 0.005）。保证河流严格贴在地形之上。
2. **带状几何**：把折线扩展为有宽度的带状面（宽度可随累积流量/级别变化），生成 `BufferGeometry`。**不穿山**（高度已采样贴合）、**无悬空**。
3. **流动发光 shader**：沿河流 UV 方向的流动光带 + 青蓝发光（`emissive` 近似），时间驱动；与水彩地形自然融合（边缘软过渡）。
4. **抗 z-fighting**：`polygonOffset` + 高度 ε 双保险。

### 6.5 大洲/标签系统（D14 仅中文 + D11 优先级视口碰撞）

**标签内容（仅中文）**：七大洲名、四大洋名、国家名（Phase 2）、（可选）主要城市（Phase 3）。

**中文数据准备**：Natural Earth 自带中文覆盖不全，需用 **ISO_A3 作为 join key**，与一份 CC0 中文名表（ISO-3166 中文）join，生成 `labels.json`（`{id, zhName, continent, lon, lat, priority, kind}`）。

**中文字体子集化（关键，D14）**：全量中文字体数 MB，必须裁剪。构建期用 `subset-font`（或 `fonttools`）提取**仅所需字形**（七大洲+四大洋+约 250 国家/地区 + UI 文案 ≈ 数百字）→ `map-zh.woff2`（数 KB ~ 数十 KB）。

**渲染（高性能，支撑数百标签 + 4K）**：

- 用 SDF 文本（非 DOM）。默认 **troika-three-text**（每标签独立 Text 实例，实现快、支撑数百标签）；若标签量上千再切换自研"共享 SDF atlas + InstancedMesh"方案（troika 不做 instancing，两者非同一技术栈，按规模二选一）。
- 标签锚点 = `project(lon,lat)` + 地形高度查询 `y`，billboard 朝向相机或固定向上。

**优先级视口碰撞剔除（D11）+ LOD 缩放联动**：

1. 每标签带 `priority`（大洲 > 大洋 > 大国 > 小国）。
2. 每帧（或节流 2~4 帧）将可见标签 3D 锚点投影到屏幕 AABB。
3. 按 priority 排序，贪心放入；与已放置 AABB 相交则隐藏（`visible=false`）。
4. **LOD 联动**：低 zoom 仅显示大洲/大洋，中 zoom 显示大国，高 zoom 显示小国——既控密度又减碰撞计算量。
5. 用简单 `Quadtree`/`RTree` 或排序列表加速（数百量级列表足够）。

**大洲标签附加**：悬浮显示、自适应缩放（随 zoom 字号微调）、标签避让（同上）。

### 6.6 相机系统（D13：固定倾斜 + 限 pan/zoom）

- **固定倾斜**：pitch 锁定 ~45°（可配 45°~60°），yaw 限制在小范围摆动（或锁定主朝向），**禁止自由翻转/进入第一人称**。
- **允许**：缩放（zoom）、平移（pan）。
- **限制 pan 边界**：限制相机目标点在世界平面 `x∈[-1.1,1.1], z∈[-0.6,0.6]` 内，防止滑出地图迷路。
- **限制 zoom**：min/max 距离，保证既有整体沙盘观感又能看清区域。
- **实现**：自定义 `SandboxControls`（基于 OrbitControls 限制，或纯自研），强制约束；阻尼平滑。

### 6.7 科技感叠加（D17：大气层辉光 + 数据标注面板）

- **大气层辉光**：在倾斜平面地图的外缘，用一个大曲率**背面弧壳**（BackSide，曲面而非整球）或**平面边缘 fresnel 边框**，配合 additive + fresnel 辉光 shader，形成边缘柔和光晕，增强科技感与深度。本作为倾斜平面沙盘（非地球仪），辉光壳应贴合平面边缘，**不使用围绕整球的球壳**。
- **数据标注面板**：hover/click 国家时，DOM 浮动卡片（drei `<Html>` 或自研 portal），显示**中文国名 + 所属大洲**（D19），定位于国家质心屏幕投影；带柔和入场动画、低饱和卡片风格，与整体美学一致。

> 未入选项（经纬全息栅格、流光扫描线）作为 Phase 3+ 可选视觉增强，预留 shader 钩子。

## 7. 资源加载方案（D16：全部打包进构建）

- 所有数据资产位于 `public/data/` 与 `public/fonts/`，**构建期打包**，运行时单次 `fetch`+解析，**离线可用、无后端**。
- **加载顺序**：先 `meta.json`（投影参数）→ 并行 `heightmap/normal/boundaries/rivers/labels` + 字体 → 解析为 typed arrays → 进场。
- **加载体验**：`Loader` 显示分项进度（drei `useProgress` 或自研），低饱和插画风加载页。
- **首屏体积优化**：
  - heightmap 用 16-bit PNG（必要时分两块懒加载）；
  - 矢量数据打包为紧凑二进制（Float32 经纬度 + UInt32 索引），远小于原始 GeoJSON；
  - 字体子集化（见 §6.5）；
  - 用 Vite 的资源 hash + 长缓存。
- **署名（合规）**：UI 角落常驻数据来源署名——Natural Earth（公共域/CC0，署名为礼貌）、Copernicus DEM（欧盟，需署名）、REMA（可选，需署名）。需提供"数据来源/许可"弹窗。

## 8. 性能优化与分级降级（D18：自动检测分档）

**目标**：桌面 60 FPS、支持 4K、数百标签、世界级数据规模。

**优化手段**：

- **Instancing / 合批**：边界线、标签、河流带状尽可能合并/实例化，减少 draw call。
- **拾取低分辨率 RT**，仅指针像素读取。
- **标签碰撞节流**（每 2~4 帧），投影复用。
- **dpr 控制**：`AdaptiveDpr`，4K 下 dpr 适当限制（避免 4× 渲染量爆显存）。
- **shader 复杂度开关**：海洋波数、水彩噪声层数、描边开关。
- **GPU 内存**：纹理用压缩/16-bit；矢量二进制直接 `BufferAttribute`。
- **Web Worker**：数据解析/标签碰撞计算放 Worker，主线程不卡。

**自动检测分档（D18）**：

| 档位 | 触发 | 配置 |
|---|---|---|
| 高 | `useDetectGPU` 评级高 + 稳定 60fps | dpr≤2、全波数、全标签、接触阴影开（shader 近似） |
| 中 | 中端 GPU 或 FPS<55 | dpr≤1.5、海洋减波、标签密度↓、接触阴影关 |
| 低 | 低端/掉帧 | dpr≤1、正弦波海洋、仅大洲标签、关闭辉光 |

- 运行时滑动窗口统计 FPS（每 ~1s 评估），跨阈值平滑切换（避免抖动），并允许手动覆盖。
- 首次加载后按设备探测初定档，之后自适应。

## 10. 边界情况与陷阱清单

| 陷阱 | 说明 | 应对 |
|---|---|---|
| 南极洲无 SRTM | SRTM 仅 ±60° | 用 Copernicus DEM（含极区）/ REMA |

| WebGL 不支持 | 极端环境 | 检测并降级到静态预览图 + 提示 |

## 12. 数据来源规划（D3：零预算全免费）

### 12.1 地形数据（MVP：GEBCO 2026 · 备选：Copernicus GLO-30）

> **MVP 实际数据源（2026-06-16 修订，见 PROGRESS Task 02b）**：改用 **GEBCO 2026 Grid（ice surface elevation）**（公共域，15″ 全球海洋陆地一体高程，equirectangular WGS84 原生投影，含 bathymetry → 喂 M2 海洋深浅渐变）。下载 Data GeoTiff（8 tiles 各 90°×90° 或单个全球文件，压缩 ~4GB），用纯 JS `geotiff` 免 GDAL 解析降采样至 4096×2048。Copernicus GLO-30 保留为更高精度（30m vs 15″≈500m）备选，Phase 4 瓦片流式时启用。

- **格式**：GeoTIFF（30m，含极区），免费、无 API、可商用（欧盟许可，需署名）。
- **下载**：Copernicus Open Access Hub / 官方 GLO-30 分发（AWS Open Data）。南极冰盖精度不足处补 **REMA**。
- **转换流程（`scripts/data-pipeline/1-dem-heightmap.mjs`）**：
  1. `gdalbuildvrt` 拼接全球 GLO-30 瓦片为虚拟 mosaic。
  2. `gdalwarp` 重投影到 equirectangular（EPSG:4326）。
  3. `gdal_translate` 降采样到目标分辨率（如 8192×4096），输出 **16-bit PNG heightmap**（高度映射到 0–65535）。
  4. `gdaldem hillshade/aspect` 或自研生成 **normal.png**。
- **前端使用**：作为顶点着色器 displacement + 片元法线贴图，只读。

### 12.2 国家边界数据 — Natural Earth（公共域/CC0）

- **格式**：Shapefile / GeoJSON。`ne_10m_admin_0_countries`（中精度）+ `ne_10m_admin_0_boundary_lines_disputed_areas`（争议）。
- **下载**：naturalearthdata.com（免费，公共域/CC0，无强制署名）。
- **转换（`2-boundaries.mjs`）**：
  1. `mapshaper` 简化（如 `-simplify 5%`）控制顶点量。
  2. `proj4` 投影到工作坐标系。
  3. `earcut` 三角化每个国家多边形（拾取/填充）+ 提取边界线段（描边）。
  4. 打包为紧凑二进制：`Float32[lon,lat...]` 顶点 + `UInt32` 偏移索引 + 属性表（ISO_A3、大洲）。
- **前端使用**：`BufferAttribute` 直接上传；颜色拾取用 ID Material。

### 12.3 河流数据 — Natural Earth Rivers

- **格式**：Shapefile/GeoJSON。`ne_10m_rivers_lake_centerlines`。
- **转换（`3-rivers.mjs`）**：简化 → 投影 → **按 heightmap 采样高度** → 带状几何生成 → 打包二进制。
- **前端使用**：流动发光 shader。

### 12.4 标签/名称数据（中文）

- Natural Earth 属性（英文/部分中文）+ **CC0 中文 ISO-3166 国名表** join（key=ISO_A3）。
- 转换（`4-labels-zh.mjs`）：生成 `labels.json`（中文国名/大洲/大洋 + 经纬度 + 优先级 + kind）。
- 前端使用：标签层渲染。

### 12.5 字体

- 来源：思源黑体 / Noto Sans SC（OFL，免费）。
- 子集化：构建期提取所需字形 → `public/fonts/map-zh.woff2`。

## 13. 开放问题 / 风险与权衡

1. **投影的"美学 vs 工程"权衡**：MVP 取 equirectangular 以快速落地，Phase 2 升级 Robinson 才真正满足"图鉴美学"。若你希望 MVP 即用 Robinson，则 Phase 1 工期 + DEM 重投影开发量需提前承担——**请确认 MVP 是否可接受 equirectangular**（默认：可接受）。
2. **高度夸张倍率**：暂定 ×2.5；可调，需实测观感（过高会失真、河流贴合更难）。
3. **标签 SDF vs DOM**：默认 SDF（性能）；若交互/可访问性优先，可对当前选中项用 DOM。**默认：SDF 为主 + 选中面板 DOM**。
4. **后处理 Bloom**：辉光优先用 fresnel shell（省性能）；若需全局泛光再引入 postprocessing，按质量档启用。
5. **WebGL2 假定**：three 0.184 默认 WebGL2；需检测极老设备并降级提示。
6. **数据体积与首屏**：全量打包首屏可能偏重，若实测 > 期望，回退到"核心打包 + 河流/详细标签懒加载"混合方案。
